## 1. Зачем

Перед введением глифов нужно подготовить поверхность карты.

Сейчас:
- Поле карты сильно затемнено (45% цвета оператора + 55% цвета фона)
- Поверх поля лежит тёмная плашка (opacity 0.18)
- Текст читается потому что он всегда белый

После введения глифов:
- Глифы идут прямо на цветное поле (плашка убирается)
- Ink выбирается по контрасту с полем (светлый/тёмный)
- Для читаемости глифа поле должно быть насыщенным, не мутным

## 2. Что меняется

### Убирается внутренняя плашка

Сейчас:
```lua
set_color(COLORS.card_back, 0.18)
rounded("fill", rect.x + 4, rect.y + 4, rect.w - 8, rect.h - 8, 8)
```

Убирается полностью. Глифы рисуются прямо на цветном split-поле.

### Затемнение поля уменьшается

Сейчас:
```lua
dark_a = lerp_color(op_color, COLORS.bg, 0.55)
-- 45% цвета оператора, 55% фона
```

Меняется на:
```lua
dark_a = lerp_color(op_color, COLORS.bg, 0.30)
-- 70% цвета оператора, 30% фона
```

Цвета операторов становятся ярче и чище.

Пример эффекта (для справки, не канон):

| Оператор | Было (0.55) | Стало (0.30) |
|----------|------------|-------------|
| FLOW (бирюзовый) | тёмный зеленовато-серый | чистый средний бирюзовый |
| CONNECT (жёлтый) | тёмный оливковый | чистый тёплый жёлтый |
| RUNTIME (графит) | почти чёрный | тёмно-серый с оттенком |

## 3. Что не меняется

- Сами значения OP_COLORS (канонические цвета операторов)
- Логика split-композиции (диагональ для minor, вертикаль для trump)
- Карты face-down (остаются тёмными)
- Цвета панелей, фона, текста
