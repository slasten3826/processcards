# Card Frame Direction

Статус:

```text
future-facing manifest direction
```

Этот документ фиксирует,
какими должны стать сами карты
как видимые объекты проекта.

Reference asset:

- [processcards_future_vision_v1.png](./vision_assets/processcards_future_vision_v1.png)

## 1. Role of manifest here

`manifest` в этом контексте отвечает не за правила,
а за:

- вид карты
- читаемую композицию face
- ощущение физического объекта
- визуальную идентичность колоды

## 2. Main card feeling

Карты должны ощущаться как:

```text
ritual technical artifacts
```

Не просто игровыми плитками
и не просто printable placeholders.

## 3. Card anatomy direction

Нужные слои карты:

- outer frame
- inner frame
- operator / identity zone
- central field
- lower status / rank / marker zone
- subtle surface texture

Это не обязано сразу собраться в layer 1,
но это направление уже фиксируется.

## 4. Front-face direction

Card face should support:

- strong silhouette
- readable operator identity
- compact but meaningful text blocks
- visual hierarchy from top to bottom

Карта должна быть читаема даже до того,
как игрок прочитает весь текст.

## 5. Back-face direction

Card back должен быть:

- технологичным
- ритуальным
- симметричным
- годным для hidden zones

Он не должен быть случайной заглушкой.

## 6. Material feeling

Желаемое ощущение поверхности:

- тёмный ламинированный картон / металл / лак
- тонкие световые акценты
- старший, не дешёвый объект

Не glossy-mobile пластик.
Не flat mockup card.

## 7. Color direction for cards

Карты могут использовать:

- family colors
- diagonal or field-based operator separation
- controlled glow accents

Но:

- цвет не должен убивать читаемость
- карта не должна превращаться в шумную радугу

## 8. Layering with gameplay

Card frame direction не должен раньше времени решать:

- точные operator payloads
- весь текст card database
- окончательную нотацию всех символов

Сначала:

- frame system
- face hierarchy
- object identity

Потом:

- full content population

## 9. What to borrow from the reference

Из [processcards_future_vision_v1.png](./vision_assets/processcards_future_vision_v1.png)
стоит брать:

- богатую рамку
- тёмную материальность
- цветовую глубину
- ощущение коллекционного объекта
- вес card face

## 10. What not to lock yet

Пока не надо жёстко фиксировать:

- exact fonts
- exact symbol placements
- final operator icon set
- final text sizes for every card family

Это следующий шаг,
после того как runtime shell и board truth стабилизируются сильнее.

## 11. Short formula

```text
cards should stop feeling like placeholders
and start feeling like ritual technical objects
```
