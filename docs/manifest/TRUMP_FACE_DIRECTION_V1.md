# Trump Face Direction V1

Статус:

```text
future-facing manifest direction
```

Этот документ нужен для будущих нас.

Он фиксирует не текущий финальный trumpet-face canon,
а правильное направление:

```text
current trump faces are acceptable prototype placeholders
final trump identity will live in a later event / arcana layer
```

## 1. Core distinction

Нужно жёстко разделять:

```text
minors = operator language
trumps = event / arcana language
```

Это значит:

- minors живут через operator pairs
- operator colors и operator glyphs — это прежде всего язык minors
- trumps не обязаны визуально говорить тем же языком как primary face identity

## 2. Table identity vs face identity

У trump cards table identity и manifest identity могут различаться по слою.

### Table layer

На table-слое trump card всё ещё определяется через:

- canonical adjacency pair
- arcana slot / trump identity

См.:

- [../table/TRUMP_IDENTITY_LAW.md](../table/TRUMP_IDENTITY_LAW.md)

### Manifest layer

На manifest-слое trump card не обязана показывать operator pair
как главный face signal.

Коротко:

```text
operator pair may define the trump in table
but it does not have to dominate the trump face in manifest
```

## 3. Current prototype status

На текущем прототипе допустимо оставить trump faces в рабочем placeholder-режиме:

- vertical split
- operator-colored glyphs
- technical pair readability
- temporary debug residue

Это acceptable only because:

- full trump presentation layer ещё не собран
- trump gameplay ещё не введён как полноценный active layer
- prototype пока проверяет общую поверхность, а не final trump spectacle

## 4. What is already known about the future

Даже если финальный trump face ещё не собран,
одна вещь уже фиксируется:

```text
the trump name is mandatory
```

Имя козыря должно быть обязательной частью его identity.

Не optional flavor,
а primary signal.

Примеры уже известных имён:

- `FOOL`
- `EJECT`
- `SHUFFLE`
- `RECAST`
- `RESET`
- `UNVEIL`
- `HALT`

## 5. Trump as event

Будущий trump layer должен читаться не как “особая пара операторов”,
а как:

```text
an event entering the machine
```

То есть козырь —
это не просто карточный face-object,
а ещё и момент появления / объявления / резолва.

Следствие:

- полная идентичность козыря раскроется не только на лице карты
- часть смысла будет жить в event presentation layer
- то, что делает козырь,
  может сообщаться в отдельном reveal / resolve moment

## 6. What this means for current design work

Сейчас не надо пытаться раньше времени собрать окончательный trump face canon.

Не нужно пока:

- изобретать final arcana sigil system
- делать полный non-operator emblem set
- насильно отрывать каждый козырь от текущего placeholder face

Сейчас достаточно:

- держать working placeholder
- помнить, что это placeholder
- не перепутать его с final law

## 7. What future trump face likely needs

Когда дойдём до следующего слоя,
trump face, вероятно, должен строиться вокруг:

- mandatory trump name
- own emblem / event mark / arcana signal
- vertical or axial ceremonial composition
- stronger singular object feeling

А не вокруг:

- two minor-style operator glyphs as the main face payload

## 8. Anti-law

Нельзя потом случайно принять временный прототипный вид за финальный закон.

То есть:

```text
current glyph-based trump face is not final canon
```

И ещё:

```text
trumps should not remain visually equivalent to minor cards with a different split
```

## 9. Short formula

```text
prototype trump face may stay technical for now
future trump face must become event / arcana identity
and trump name is mandatory
```
