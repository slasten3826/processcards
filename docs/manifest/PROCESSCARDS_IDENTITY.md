# ProcessCards Identity

Статус:

```text
canonical project identity note

```

## 1. What this game is not

ProcessCards is not primarily a board-building card game.

The player does not gradually construct a personal field
and then command it directly.

## 2. What this game is

ProcessCards is a game about intervening in an already moving machine.

The world has an open layer and a hidden supporting layer.
The player does not create this motion.
The player enters it.

## 3. World reading

`manifest` and `latent` are not just zones.

They are the breathing surface of the world:

- open layer
- hidden supporting layer
- consumption
- ascent
- refill

The world updates itself before the played impulse resolves.

## 4. Hand reading

`hand` is not a place of future board-objects.

`hand` is a reservoir of interventions.

A card from hand is not primarily a permanent thing.
It is a packet of possible effects.

## 5. Turn reading

A turn is not “place a card onto the table”.

A turn is:

- commit one visible world-node
- consume it
- let the hidden layer rise
- let the future refill
- then apply one chosen operator effect

So the move is not direct control.
It is delayed intervention into an updating structure.

## 6. Information reading

The game does not only distinguish between hidden and open.

It also distinguishes between:

- hidden
- known
- revealed

This means the player may know something
that the world has not yet manifested publicly.

## 7. Short formula

ProcessCards is not about controlling a board.

It is about sending interventions
into a breathing world-strip
that becomes alive only while it is being played.
