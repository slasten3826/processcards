# CYCLE

Symbol:

```text
☲
```

Статус:

```text
canonical operator family
```

## 1. Core identity

`☲` = exchange through turnover.

Short formula:

```text
take one / release one
```

## 2. What it is not

`☲` is not:

- pure draw
- pure discard
- reveal
- removal from field by default

## 3. Weak law

Weak `☲` performs a narrow cycle action.

Current safe reading:

- draw a card
- then discard a card

This is a provisional but stable working direction.

## 4. Strong law

Strong `☲` widens cycle value.

Strong baseline still applies:

```text
draw 2
```

The card reading then opens broader exchange logic,
not just raw card count.

## 5. Pair law directions

- `☲☳` — choose what to keep or what to release
- `☲☴` — observe before cycle choice
- `☲☵` — cycle through encoded ordering
- `☲☶` — cycle while ignoring one native cycle restriction
- `☲☱` — install cycle as runtime-form churn

## 6. Physical execution law

Cycle must be executable with physical cards.

That means:

- actual draw from deck
- actual discard to grave
- no hidden engine-side replacement without visible card movement

## 7. Restrictions

- cycle should not become free tutor
- discard destination must stay explicit
- cycle must preserve bounded information access

## 8. Open questions

- whether weak `☲` is always `draw 1 discard 1`
- whether strong `☲` may include keep-one-from-many patterns
- exact runtime-form cadence for `☱☲`

## 9. Short formula

```text
☲ CYCLE = controlled turnover through draw/discard exchange
```
