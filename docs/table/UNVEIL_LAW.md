# Unveil Law

Статус:

```text
canonical table law
adopted individual trump event
```

Legacy breadcrumb:

```text
XVIII Moon
```

Core edge:

```text
△ -> ☲
MANIFEST -> CYCLE
```

## 1. Core identity

`UNVEIL` is a total visibility trump.

It does not primarily move cards.
It changes the visibility state of the machine.

Short formula:

```text
hidden state -> visible state
```

## 2. Reading

`△ MANIFEST` here is not local reveal.

It is global exposure of what was hidden.

`☲ CYCLE` makes that exposure propagate across the machine,
so hidden structures do not stay private exceptions.

So this trump reads as:

```text
manifestation expanded into system-wide visibility
```

Not:

```text
peek
```

And not:

```text
temporary scouting
```

## 3. Resolution scope

When `UNVEIL` becomes known,
reveal all cards that are face-down
at the moment this trump resolves.

This includes:

- face-down cards in the latent row of manifest chain
- face-down cards in target zone
- the top card of deck, if it is face-down
- any other face-down card created by future lawful effects

This does not include:

- hand cards, because hand is not a face-down zone under current rules
- hidden runtime, because hidden runtime does not exist under current rules

Short formula:

```text
all current face-down cards become revealed
```

## 4. Trump consequence

`UNVEIL` may expose other trumps.

If a newly revealed card is a trump
outside `targets`,
that trump enters ordinary trump event flow.

In current machine language:

1. the card becomes `revealed`
2. it becomes `known`
3. if it is outside `targets`, it enters ordinary trump event flow
4. it goes into `trump flow`

So yes:

```text
UNVEIL may send newly exposed trumps into trump flow
```

## 5. Target zone rule

If a card is revealed in `targets`:

- if it is a trump, it remains in `targets`
- if it is not a trump, discard it to grave immediately

This discard happens during the same resolution.

Short formula:

```text
target non-trump revealed -> grave
target trump revealed -> remains installed
```

This is the compiler-zone override.

## 6. Persistence rule

Cards revealed by `UNVEIL` remain revealed.

This trump does not create temporary visibility.
It changes actual visible state.

Short formula:

```text
revealed means revealed
```

## 7. Topdeck rule

If the top card of deck is face-down,
reveal it.

This trump does not draw that card
and does not move it by default.

It only makes the top of deck visible.

Short formula:

```text
topdeck may become manifest without being drawn
```

## 8. Gameplay role

This trump:

- opens the latent row
- exposes target construction
- reveals future pressure through topdeck visibility
- reduces the exclusive information advantage of `OBSERVE`
- shifts play from hidden inference toward open structural reading

This is not a reset trump.
This is not a board rewrite trump.

This is:

```text
global visibility pressure
```

## 9. Relation to general trump law

`UNVEIL` does not invent a separate ecology.

It uses:

- ordinary ordered resolution
- ordinary trump flow
- ordinary chain close handling
- ordinary trump-zone law

Its specificity is not in ecology.
Its specificity is in changing visibility state
across the machine.

## 10. Design character

This trump should feel:

- exposing
- irreversible
- clarifying
- destabilizing for hidden plans
- less violent than `RESET`, but broader than `OBSERVE`

It does not destroy structure directly.
It removes secrecy as protection.

## 11. Minimal canonical text

```text
Reveal all face-down cards.
If a non-trump card is revealed in target zone, discard it immediately.
Revealed trump cards in target zone remain.
Newly revealed trumps outside target zone enter ordinary trump event flow.
```
