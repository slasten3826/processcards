# Repeat Law

Статус:

```text
canonical table law
adopted individual trump event
```

Legacy breadcrumb:

```text
XVII Star
```

Core edge:

```text
☱ -> ☲
RUNTIME -> CYCLE
```

## 1. Core identity

`REPEAT` is a chain-deepening trump.

It does not invent a fresh law.
It does not replay some vague remembered event.
It repeats the direct trump effect
that caused `REPEAT` to resolve.

Short formula:

```text
repeat the direct parent trump effect
or do nothing
```

## 2. Reading

`☱` here is not installation.

It means the already-entered trump law
is still held in machine reality
as the immediate active cause.

`☲` here is not ordinary hand cycle.

It means that already-entered disturbance
passes once more through the machine.

So this trump reads as:

```text
the immediate trump disturbance enters once again
```

This is not:

```text
copy the last thing
```

And not:

```text
recall some distant past event
```

It is:

```text
repeat the direct causing trump effect
```

## 3. Parent trump rule

`REPEAT` tracks only the direct trump
that caused `REPEAT` to become known.

That trump is the:

```text
parent trump
```

So the causal line is:

1. trump `A` resolves
2. `A` causes `REPEAT`
3. `REPEAT` resolves
4. `REPEAT` repeats effect `A`

This keeps the card:

- local
- causal
- readable

## 4. Resolution

When `REPEAT` becomes known:

1. check whether `REPEAT` was caused by a direct parent trump
2. if no direct parent trump exists, `REPEAT` has no effect
3. if a direct parent trump exists, repeat that parent trump's effect
4. any newly exposed trumps enter ordinary trump event flow
5. after `REPEAT` fully resolves, `REPEAT` itself follows ordinary trump ecology

Short formula:

```text
parent trump exists -> repeat its effect
no parent trump -> no effect
REPEAT then follows ordinary trump ecology
```

## 5. Scope restriction

`REPEAT` does not:

- copy ordinary minor effects
- repeat some vague previous event
- search the whole past chain for a target
- bypass ordinary trump flow
- suppress ordinary chain-close handling

It repeats only:

```text
the direct causing trump effect
```

## 6. Relation to trump ecology

`REPEAT` does not invent separate ecology.

It uses:

- ordinary parent-cause tracking
- ordinary trump flow
- ordinary ordered resolution
- ordinary chain-close handling
- ordinary trump-zone law after its own resolution

Its specificity is not in escaping the machine.

Its specificity is in sending
the immediate trump disturbance
through one more cycle of the same machine.

## 7. Why this works

`REPEAT` is empty outside living trump turbulence.

If no trump directly caused it,
it does nothing.

If it appears inside an active trump flow,
it can deepen that flow sharply
without inventing a new class of action.

That makes it:

- contextual
- recursive
- nonlinear
- dangerous in the right place

## 8. Minimal canonical text

```text
If REPEAT was caused by a direct parent trump,
repeat that trump's effect.
Otherwise REPEAT has no effect.
After REPEAT fully resolves,
it follows ordinary trump ecology.
```
