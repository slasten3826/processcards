## 1. Core Identity

`ENOUGH` is a sufficiency trump.

It does not create value.
It does not force a board rewrite.
It does not overpower the machine.

It checks whether the current relation
is already sufficient for victory.

Short formula:

```text
seen outside == held inside
```

Or:

```text
the machine observes the table
and finds that the player already holds enough
```

---

## 2. Reading

`‚ėī OBSERVE` here is not ordinary reveal.

It is a count of what is currently seen.

`‚ėĪ RUNTIME` here is not installation.

It is a check against what is already held
inside the player's active state.

So this trump reads as:

```text
outer visibility
is measured
against inner capacity
```

This is not:

```text
moderation as virtue
balance as vague aesthetics
mixing as symbolism
```

It is executable proportion.

---

## 3. Core Law

When `ENOUGH` becomes known:

1. Complete ordinary repair first.
2. Count all currently revealed cards in `manifest chain`.
3. Count all currently revealed cards in `latent layer`.
4. Count cards in hand.
5. If revealed board count equals hand count, win immediately.
6. Otherwise, `ENOUGH` follows ordinary trump ecology.

Short formula:

```text
repair
count seen
count held
if equal -> win
```

---

## 4. What Counts

Count only currently revealed cards in:

- `manifest chain`
- `latent layer`

Do not count:

- `target zone`
- `grave`
- `deck`
- `hand`
- `play zone`
- `runtime`
- `trump zone`
- `trump flow`
- hidden / face-down cards

Normal visible-board range:

```text
6 revealed manifest
+ 0..6 revealed latent
= 6..12 visible board
```

So `ENOUGH` is not simply:

```text
have 6 cards in hand
```

It is:

```text
hold exactly as much
as the board currently reveals
```

---

## 5. Repair Timing

Repair must happen before the check.

This is important.

`ENOUGH` does not judge a torn intermediate state.
It judges the stabilized board
after ordinary repair has completed.

If `ENOUGH` emerges from latent:

1. `ENOUGH` leaves the latent slot into trump resolution.
2. Ordinary repair fills the slot.
3. The repaired board is then counted.

Short formula:

```text
ENOUGH checks stabilized state
not reveal-moment turbulence
```

---

## 6. Why This Works

`ENOUGH` is strong because it offers
a non-forceful exit.

It does not:

- draw
- discard
- move cards
- attack the board
- complete the normal target trace

It says:

```text
the current relation is already sufficient
```

That makes it a true alternate victory trump.

Not:

```text
push harder
```

But:

```text
stop
take the exit
```

---

## 7. Relation to Hand Size

There is currently no hand limit.

This matters.

`ENOUGH` can care about exact hand count
without requiring discard-down cleanup.

Large hands are possible.
That does not break the card.

It simply means:

```text
more held
requires more seen
```

This creates strong ecology with:

- `RUSH`
- `RESET`
- reveal-heavy play
- repair timing
- latent visibility

`ENOUGH` rewards exact relation,
not maximum hand inflation.

---

## 8. Revealed Latent Layer

The revealed latent layer matters strongly.

This is intentional.

If latent is mostly hidden:

```text
ENOUGH usually checks against 6
```

If latent is heavily revealed:

```text
ENOUGH may check against 7..12
```

So increased visibility is not automatically good.

It changes the required held threshold.

Short formula:

```text
more seen
requires more held
```

---

## 9. Victory Timing

If `ENOUGH` succeeds,
the game should end immediately.

Preferred rule:

```text
ENOUGH victory interrupts the rest of the chain
```

Reason:

```text
Enough means enough
```

If the machine has already found sufficiency,
the game should not ask for one more proof.

## 10. Design Character

`ENOUGH` should feel:

- exact
- quiet
- final
- nonviolent
- verdict-like
- rare as true victory
- more about recognition than force

It should not feel:

- explosive
- dominant
- greedy
- procedural for its own sake
- legacy-coded for tarot fans first

Its power is not in action.

Its power is in closure.
