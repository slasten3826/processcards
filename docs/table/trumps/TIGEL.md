## 1. Core Identity

`TIGEL` is a chamber trump.

It does not create immediate value.
It changes how much trump-event pressure
the machine can hold.

Short formula:

```text
empty chamber -> larger chamber
```

Or:

```text
hold more unresolved trump-history
```

---

## 2. Reading

```text
hold
```

`☵ ENCODE` here is not ordinary inscription.

```text

what was only empty capacity
```

## 3. Core Law

When `TIGEL` enters `trump zone`:

1. `TIGEL` anchors there.
2. While `TIGEL` remains anchored, double the current `trump zone` capacity.
3. `TIGEL` occupies one slot inside that doubled capacity.
4. Ordinary release / overflow does not remove `TIGEL`.
5. Only an effect that explicitly targets a trump in `trump zone` may remove `TIGEL`.

Short formula:

```text
anchor in trump zone
double chamber capacity
ignore ordinary release
```

---

## 4. Capacity Law

`TIGEL` does not add a fixed bonus.

It doubles the current base chamber.

Formula:

```text
base_capacity = B
with TIGEL anchored:
  total_capacity = B * 2
  working_capacity_for_other_trumps = (B * 2) - 1
```

Normal board:

```text
B = 2
TIGEL anchored -> total capacity 4
free working slots = 3
```

Circle 7:

```text
B = 3
TIGEL anchored -> total capacity 6
free working slots = 5
```

This scaling is important.

`TIGEL` should read as:

```text
chamber multiplier
```

Not:

```text
small bonus slot
```

---

## 5. Anchor Timing

`TIGEL` does not become active while still inside trump flow.

It becomes active only when it actually enters `trump zone`.

Short formula:

```text
trump flow = resolving event
trump zone = persistent chamber
```

No doubled capacity exists
until the card is parked and anchored.

---

## 6. Repeat Law

`TIGEL` should be repeatable,
but not stack with itself.

If `REPEAT` repeats `TIGEL`:

- no duplicate `TIGEL` is created
- no second chamber multiplier is created
- the same anchor state is re-run

Short formula:

```text
TIGEL repeat = idempotent
```

If `TIGEL` is not anchored yet:

```text
repeat resolves the same future anchor once
```

If `TIGEL` is already anchored:

```text
repeat does not create another doubling
```

This preserves causal repeat
without cloning chamber structure.

---

## 7. Circle 7 Interaction

Circle 7 changes `☲ CYCLE` into stored-force repetition:

```text
☲ repeats the latest trump
currently stored in trump zone
```

This makes `TIGEL` especially dangerous there,
because `TIGEL` is not just more storage.
It is more storage
inside a floor that re-fires stored trump pressure.

Short formula:

```text
TIGEL on Circle 7 = enlarged chamber
for repeated stored force
```

---

## 8. TIGEL As Circle 7 Target

If `☲` on Circle 7 targets `TIGEL` itself,
the chamber should not stay abstractly doubled.

It should collapse and re-anchor.

Recommended handling:

1. Remove `TIGEL` from `trump zone`.
2. Recalculate `trump zone` capacity without `TIGEL`.
3. Perform ordinary overflow / release until the zone fits.
4. Return `TIGEL` to `trump zone`.
5. Reapply doubled capacity.

Short formula:

```text
☲ on TIGEL = chamber collapse and re-anchoring
```

This means `TIGEL` can function as a violent chamber reset:

```text
stored chamber
-> collapse
-> forced release
-> chamber restored
```

This is strong,
but process-consistent.

It also prevents the Circle 7 interaction
from degenerating into fake infinite chamber stability.

## 9. Why This Works

`TIGEL` is strong because it changes event capacity,
not because it gives direct board value.

It does not:

- draw
- reveal
- kill
- swap
- rewrite board state immediately

It changes how much trump-history
the machine can keep alive.

That makes it a real `IX` card:

- quiet
- delayed
- infrastructural
- strange
- not immediately profitable
- highly dangerous in the right chamber

`TIGEL` should feel:

The point is not theme.
