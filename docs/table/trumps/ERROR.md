## 1. Core Identity

`ERROR` is a grave-to-hand overload trump.

It does not discard.
It does not clean the hand.
It does not ask what the player wants back.

It opens the whole `grave`
and admits all of it into `hand`.

Short formula:

```text
grave opens
all residue returns
nothing returned may be refused
```

---

## 2. Reading

`☳ CHOOSE` here is not deliberate player preference.

It is forced collapse of separation:

```text
what had been cut away
is no longer allowed to stay away
```

`☱ RUNTIME` here is not support.

It is the active burden now forced back inside:

```text
the state must carry all of it
```

So this trump reads as:

```text
what had already been cut away
collapses all at once
back into active possession
```

This is not:

```text
recovery
mercy
careful recursion
```

It is compulsory re-admission.

---

## 3. Core Law

When `ERROR` becomes known:

1. Move all cards from `grave` into `hand`.
2. Preserve their grave order during transfer.
3. `grave` becomes empty.
4. `ERROR` itself does not enter `hand`.
5. After `ERROR` fully resolves, it follows ordinary trump ecology.

Short formula:

```text
grave -> hand
all of it
no refusal
ERROR -> ordinary trump ecology
```

---

## 4. No Refusal

This law is load-bearing.

The player may not:

- choose only some grave cards
- leave grave cards behind
- redirect returned cards elsewhere
- convert return into discard

If `ERROR` resolves,
the whole residue load returns.

Short formula:

```text
return is mandatory
```

---

## 5. Why This Is Dangerous

`ERROR` looks like power.

It can also be burden.

Because there is currently no hand limit
and no free discard outlet,
the returned grave may become:

- overload
- dead weight
- delayed self-destruction
- fuel that the player cannot metabolize safely

This is intentional.

`ERROR` does not give freedom.
It gives possession without release.

Short formula:

```text
returned residue becomes renewed bondage
```

---

## 6. Why This Works

`ERROR` is strong not because it restores value,
but because it destroys the boundary
between dead residue and live burden.

Everything that had already fallen out of play
is suddenly back in the active interior.

That makes it a true devil-slot event:

- accumulation without mercy
- access without freedom
- return without cleansing

It is not temptation.
It is lock-in through abundance.

Or more precisely:

```text
separation fails
and runtime inherits the whole residue load
```

---

## 7. Relation to MAXIMIZE

`ERROR` pairs dangerously with `MAXIMIZE`.

`ERROR` can stuff `hand`
with everything that had already died.

`MAXIMIZE` can then spend that hand
directly into `manifest`.

This pairing is not safety.
It is escalation:

```text
ERROR restores the burden
MAXIMIZE spends it recklessly
```

That is not a combo exception.
It is ordinary ecology revealing structural excess.

---

## 8. Relation to ENOUGH

`ERROR` can destroy `ENOUGH` alignment
by inflating `hand` far beyond visible board count.

It can also create strange later alignment
if the player survives long enough
to metabolize the returned residue.

So `ERROR` does not merely add cards.
It changes the entire relation
between held burden and visible world.

---

## 9. Relation to SHUFFLE

`SHUFFLE` recirculates residue into `deck`.

`ERROR` does the opposite:

```text
residue bypasses future circulation
and returns directly into hand
```

So:

- `SHUFFLE` delays residue into future uncertainty
- `ERROR` restores residue as immediate burden

This is a real polarity.

---

## 10. Design Character

`ERROR` should feel:

- overloaded
- compulsory
- binding
- excessive
- richer and worse at the same time
- more dangerous than it first appears

It should not feel:

- generous
- cleansing
- like a simple reward
- like grave recursion under control

It is returned excess without release.
