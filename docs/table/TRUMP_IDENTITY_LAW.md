# Trump Identity Law

Статус:

```text
canonical table law
```

Этот документ фиксирует,
что именно считается полной trump-колодой `ProcessCards`
на уровне identity.

Это закон про:

- состав `22` trump cards
- canonical operator pairs
- already known working names

Это не документ про:

- полную механику всех козырей
- runtime behavior
- setup placement rules

## 1. Core law

Козыри `ProcessCards` строятся так:

```text
trumps = all unique adjacency pairs of the canonical topology
```

То есть:

- берутся только adjacency-пары канона
- self-pairs не входят
- порядок не удваивается

Значит:

```text
AB = BA
```

внутри trump identity.

Итого:

```text
22 trump cards
```

## 2. Difference from minors

Важно различать:

### Minors

- `10 x 10` ordered physical grid
- `AB` и `BA` физически существуют как разные cards

### Trumps

- only unique adjacency pairs
- `AB = BA`
- no self-pairs
- no duplicate reverse-order identity

Коротко:

```text
minors = ordered operator grid
trumps = unordered topology adjacencies
```

## 6. Temporary runtime naming

Пока все `22` названия ещё не собраны,
в runtime допустим временный слой:

- `TRUMP-1`
- `TRUMP-2`
- ...
- `TRUMP-22`

Но:

```text
TRUMP-1 .. TRUMP-22 must map to the real canonical 22-pair list
```

То есть временный id допустим,
но random/cyclic fake pair generation уже недопустим.
