# Unbound Law

Статус:

```text
canonical table law
adopted individual trump event
```

Legacy breadcrumb:

```text
VI Lovers
```

Core edge:

```text
☷ -> ☴
DISSOLVE -> OBSERVE
```

## 1. Core identity

`UNBOUND` is a target-slot displacement trump.

It does not merely destroy.
It does not merely swap.
It dissolves one current target-line
and takes that exact slot.

Short formula:

```text
dissolve one target card
UNBOUND takes that exact slot
```

## 2. Reading

`☷` here is not broad field removal.

It means one current compiled line
is cut out of its binding place.

`☴` here is not private inspection.

It means one currently occupied target-line
is singled out and chosen.

So this trump reads as:

```text
one fate-line is unbound
another takes its place
```

This is not:

```text
global destruction
```

And not:

```text
random replacement
```

It is:

```text
chosen compiler-slot replacement
```

## 3. Core law

When `UNBOUND` becomes known:

1. choose any currently occupied card in `targets`
2. remove that card from its exact target slot
3. place `UNBOUND` into that exact slot
4. if the removed card is a trump, move it to `trump zone`
5. if the removed card is not a trump, move it to `grave`

Short formula:

```text
target card dissolved
UNBOUND takes its slot
dissolved trump -> trump zone
dissolved non-trump -> grave
```

## 4. Target scope

`UNBOUND` acts on target occupancy,
not on visibility class.

Legal target cards in `targets` may be:

- trump
- non-trump
- hidden
- known
- revealed

This trump does not require extra reveal wording.
Its legal scope is defined by:

- slot occupancy
- membership in `targets`

Short formula:

```text
UNBOUND acts on occupied target slots
not on face-state class
```

## 5. Exact slot consequence

`UNBOUND` does not enter `targets` freely.

It inherits the exact slot
of the dissolved target card.

This is important.

It means the card is:

```text
slot replacement
```

not:

```text
free target insertion
```

## 6. Routing rule

The dissolved card does not re-resolve.

If it is a trump,
it becomes open trump residue:

```text
dissolved trump -> trump zone
```

If it is not a trump,
it becomes ordinary residue:

```text
dissolved non-trump -> grave
```

This avoids accidental extra trump-trigger noise
and keeps the compiler rewrite local.

## 7. Relation to target law

`UNBOUND` is a special compiler override.

It does not follow ordinary target reveal routing.

Ordinarily:

- revealed non-trump target -> grave + deck refill
- target trump stays face-up in `targets`

But under `UNBOUND`:

- chosen target occupancy is dissolved directly
- `UNBOUND` takes that exact slot
- removed card routes by class as stated above

So this trump does not reveal and then route.
It replaces target occupancy directly.

## 8. Relation to trump ecology

`UNBOUND` does not invent separate ecology.

It uses:

- ordinary target-zone access rules
- ordinary class routing after dissolution
- ordinary trump-zone law for any dissolved trump
- ordinary grave law for any dissolved non-trump
- ordinary chain-close handling for `UNBOUND` itself

Its specificity is not in ecology.

Its specificity is in rewriting
one compiler slot directly.

## 9. Restrictions

`UNBOUND` does not:

- create a free new target slot
- dissolve multiple target slots at once
- re-trigger the dissolved target card
- send the dissolved trump into `trump flow`
- affect ordinary non-target zones by default

It is local to:

```text
one occupied compiler slot
```

## 10. Design character

`UNBOUND` should feel:

- precise
- local
- compiler-deep
- disruptive
- stronger when targets already matter
- less explosive than `RESET`
- more surgical than `SWAP`

It does not rewrite the whole target structure.
It cuts one bound line
and installs another.

## 11. Minimal canonical text

```text
Choose one occupied card in target zone.
Remove it from that exact slot.
Place UNBOUND into that slot.
If the removed card is a trump, move it to trump zone.
Otherwise move it to grave.
```
