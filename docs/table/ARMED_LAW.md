# ARMED LAW

## Status

Current table law candidate.

This law defines the shared meaning of `armed`
across the machine.

## Core statement

`armed` is a pre-manifest state.

It means:

```text
selected
visible
not yet executed
```

An armed object is not yet consumed by the machine.

It is the currently prepared choice
that still waits for manifestation through `△`.

## Manifest relation

Short formula:

```text
armed = pre-manifest
△ = manifest the armed choice
```

So the player interaction rhythm becomes:

1. choose
2. see the choice
3. confirm it with `△`

Only then does the machine proceed.

## Why this matters

Without a real armed phase,
the machine collapses too quickly from:

```text
possible
-> executed
```

That is too abrupt for the main interaction loop.

The machine should instead read as:

```text
possible
-> armed
-> manifested
```

## Armed is not execution

An armed object is:

- not yet resolved
- not yet spent
- not yet committed to consequence

It is only the currently prepared object of will.

## Current armed families

The machine already needs three armed families:

### 1. Armed hand

```text
this is the hand card prepared for the turn
```

### 2. Armed operator

```text
this is the operator prepared for confirmation
```

### 3. Armed target

```text
this is the effect target prepared for confirmation
```

These are different machine states,
but they share the same pre-manifest meaning.

## Visual consequence

If something is armed,
the player must be able to see that:

- it is the current selected object
- it is not merely legal
- it is not yet manifested

So `armed` must have its own visible language,
different from:

- legal target trains
- ordinary idle state
- already resolved state

## Button consequence

`△` should be read as:

```text
manifest the current armed choice
```

not merely:

- next
- continue
- generic submit

It is specifically the visible permission
for the machine to turn armed state into action.

## Scope

This law should govern:

- armed hand in turn launch
- armed operator in operator phase
- armed target in target phases

and any future branch where the machine requires:

```text
visible selection before visible execution
```

## Short formula

```text
armed is the machine's pre-manifest state.
it means the player has chosen,
the machine has not yet acted,
and `△` is the act of manifestation.
```
