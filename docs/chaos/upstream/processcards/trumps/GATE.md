## 1. Core Identity

`GATE` is a gatekeeping trump.

It does not create value.
It does not purge at random.
It does not touch the hidden board directly.

It divides the visible board
by one chosen key.

Short formula:

```text
choose one visible key
check all visible cards
shared operator -> stay
no shared operator -> grave
```

## 3. Core Law

When `GATE` becomes known:

1. Choose one revealed card in `manifest chain` as `gatekeeper`.
2. Check each revealed card in `manifest chain`.
3. If a revealed card shares at least one operator with the gatekeeper, it remains in place.
4. If a revealed card shares no operator with the gatekeeper, send it to `grave`.
5. Check each revealed card in `latent layer` the same way.
6. Check the revealed top card of `deck`, if any, the same way.
7. Hidden latent cards are not checked.
8. After all purge completes, resolve ordinary board repair.
9. After `GATE` fully resolves, it follows ordinary trump ecology.

Short formula:

```text
choose gatekeeper
check all visible cards
shared -> stay
no shared -> grave
repair
```

## 4. Gatekeeper Law

The `gatekeeper` is:

- one revealed card in `manifest chain`
- the source of the gate condition
- guaranteed to survive

- is not itself re-evaluated as a failing subject

Short formula:

```text

```

## 5. Visible Scope

`GATE` acts only on currently visible board state.

Included:

- `manifest chain`
- revealed cards in `latent layer`
- revealed top card of `deck`, if any

Excluded:

- hidden latent cards
- `hand`
- `grave`
- `target zone`
- `runtime`
- `trump zone`
- `trump flow`

This matters.

`GATE` does not penetrate concealment.
It divides only what is already exposed to the sun.

---

## 6. Repair After Purge

Repair happens only after all checks are complete
and all failing visible cards have already gone to `grave`.

There is no mid-purge refill.

Then ordinary board repair resolves:

```text
manifest gaps are repaired from their columns
latent gaps refill from deck
```

So `GATE` does not create a special repair ecology.
It uses ordinary post-damage repair order.

---

## 7. Why This Works

`GATE` is strong because it restructures
the visible world along one operator relation.

It does not:

- rewrite the hidden board
- attack hand state
- invent a new zone
- create new geometry

It says:

```text
everything visible is now tested
against one visible law
```

That makes it a true structural division trump.

---

## 8. Relation to Reveal Density

`GATE` becomes stronger
when more of the board is revealed.

If latent is mostly hidden,
`GATE` behaves mainly as a `manifest` filter.

If latent is heavily revealed,
`GATE` can purge much more aggressively.

This is intentional.

`GATE` rewards not raw force,
but correct timing of visible-state division.
