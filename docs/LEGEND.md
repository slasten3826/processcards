# Легенда ProcessCards

Это не rulesheet.
Это хроника пути идеи.

Здесь можно писать как летопись, сказку, технический дневник или найденный манускрипт.
Главное - сохранять хронологию: какой ход мысли к чему привёл, что было отброшено, что стало законом, что осталось открытым.

## I. Колода без войны

Сначала была не игра, а ощущение: в карточных играх есть особое удовольствие сборки двигателя.
Не победа над противником.
Не гонка урона.
Не армия существ на столе.

А именно:

```text
карты входят в зоны
зоны меняют смысл карт
последовательность действий создаёт машину
grave становится историей
deck становится будущим
hand становится поверхностью выбора
```

В `Magic: The Gathering` это удовольствие часто прячется внутри combo deck.
Но combo deck тащит solitaire engine в duel shell: второй игрок вроде есть, но его присутствие становится нарушенным.

Так появилась первая развилка:

```text
оставить engine
убрать duel
```

Из этой развилки родилась форма:

```text
ProcessCards = solitaire engine честно, без притворства дуэли
```

## II. Canon становится колодой

Потом пришёл ProcessLang canon.

Не как тема и не как украшение.
Canon дал операторную топологию:

```text
▽ FLOW
☰ CONNECT
☷ DISSOLVE
☵ ENCODE
☳ CHOOSE
☴ OBSERVE
☲ CYCLE
☶ LOGIC
☱ RUNTIME
△ MANIFEST
```

Minor-карта стала directed operator pair:

```text
NUMBER[SUIT]
```

Так из 10 операторов вышло 100 minor-карт.
Направление стало важным: `FLOW[LOGIC]` не равен `LOGIC[FLOW]`.

Козыри возникли как edges canon topology.
Их должно быть 22.
Но к этому моменту стало ясно: козырь не должен быть просто сильной картой.

## III. Закрытый мир

Следующий закон был суровым:

```text
122 cards
one deck
no external tokens
no creatures
no enemies
no allies
no mana
no HP
```

Если эффект требует внешний объект, счётчик, жетон или отдельную экономику, он не принадлежит core ontology.

В игре остаются только:

- положение карт;
- порядок карт;
- открытость и закрытость;
- зоны;
- relation checks;
- локальные rule bends;
- история текущего действия.

Это сделало игру закрытой системой.
И одновременно заставило каждую механику отвечать за себя.

## IV. Manifest chain получает скрытую спину

Долгое время manifest chain была скорее абстрактной строкой состояния.
Потом она получила физическую форму:

```text
5 columns x 2 layers
```

Верхний слой стал `manifest row`: пять видимых активных карт.
Нижний слой стал `latent row`: пять скрытых карт, уже принадлежащих столу.

Так появилась важная мысль:

```text
hidden does not mean unformed
```

Latent - не вторая рука и не бесформенный запас.
Это структурированная скрытая сторона текущего процесса.

Когда manifest card уходит из slot, latent card из той же колонки поднимается.
Не любая скрытая карта.
Именно локальная.

Так board получил память места.

## V. Minor-карты начинают держать игру

Когда стол стал 5x2, minor-карты перестали быть просто набором символов.
Они стали состояниями.

Между minor-картами появились три отношения:

```text
weak
strong
mirror
```

Weak держится на совпадениях операторов.
Strong проверяется через adjacency из canon.
Mirror остаётся особым отражением пары.

Здесь игра начала становиться играбельной без козырей.
И это важно: почти вся основная игра должна работать на minor-слое.

Козыри не должны спасать слабое ядро.
Они должны тревожить уже работающую машину.

## VI. Targets перестают быть тремя целями

Targets сначала могли читаться как отдельные goals.
Потом это стало слишком плоско.

Сильная текущая формула:

```text
targets compile one victory grammar
```

Не цель A, цель B, цель C.
А три слота, которые вместе составляют грамматику победы.

Пример текущего направления:

```text
slot 1 = scope
slot 2 = pattern
slot 3 = resolution / modifier
```

Это ещё не финальная система.
Но сама мысль важна: игрок строит не только board, но и будущее условие успеха.

## VII. Козырь отказывается быть картой состояния

Когда minor-карты были названы state, козыри получили противоположный закон:

```text
minor = state
trump = event
```

Козырь не должен спокойно лежать в руке как обычная карта.
Он не должен быть passive buff.
Он не должен уходить в grave как простой spent card.

Козырь хочет случиться.

Если trump становится известен, он разрешается.
Даже observe hidden trump становится опасным контактом.

Так появилась trump zone:

```text
two-event pressure chamber
```

Первый resolved trump - событие.
Второй - камера заряжена.
Третий - reset pulse: два старых trump и текущий возвращаются в deck, а trump zone очищается.

## VIII. Первый козырь: RECAST

Из 22 козырей пока механически проработан один:

```text
RECAST = ☳ -> ☶
CHOOSE -> LOGIC
```

Его смысл:

```text
choice suspended into structural reinterpretation
```

Он меняет местами мир, скрытую сторону и поверхность вмешательства:

```text
old manifest -> new hand
old latent -> manifest
old hand -> latent
old hand overflow -> grave
trump -> trump zone
```

RECAST важен не только как карта.
Он проверяет, выдерживает ли board форму после сильного события.

Если после него остаются:

```text
5 manifest slots
5 latent slots
ordered grave
valid hand
valid trump zone
```

значит core loop уже способен пережить топологическое потрясение.

## IX. Lua как рабочий материал

Когда natural-language design начал разрастаться, стало ясно: нужна исполняемая форма.

Lua выбран не случайно.

Он подходит этой игре потому что:

- правила можно держать в простых tables;
- поведение легко менять;
- state можно читать глазами;
- прототип не требует тяжёлой инфраструктуры;
- можно быстро перейти от rules engine к playable loop;
- позже можно сделать GUI, не выбрасывая ядро.

Текущий путь:

```text
rules engine -> tests -> CLI -> functional GUI
```

CLI нужен, чтобы быстрее начать играть.
GUI нужен, потому что игра просит руки: карты хочется тыкать мышкой, двигать, выбирать, читать зонами.

Идеальный ранний GUI не обязан быть красивым.
Он должен быть функциональным карточным столом.

## X. Где мы стоим сейчас

На текущем рубеже есть:

- игра на 122 карты;
- 100 minor-карт как почти готовое ядро;
- 22 trump slots как будущая event ecology;
- один проработанный trump: `RECAST`;
- board model `5 columns x 2 layers`;
- relation system для minor-карт;
- target compiler как направление;
- желание сделать первый playable prototype.

Следующий настоящий шаг:

```text
lua tests/run.lua
```

А потом:

```text
lua src/cli.lua
```

Когда игра впервые станет playable, легенда продолжится уже не только из документов, но из сопротивления самой машины.

machines only. not for humans.
